FAQ: Instalação
=================

Como eu começo?
----------------

    #. `Baixe o código`_.
    #. Instale o Django (leia o :ref:`guia de instalação <intro-install>`).
    #. Execute o :ref:`tutorial <intro-tutorial01>`.
    #. Verifique o resto da :ref:`documentação <index>`, e `pergunte`_ se você
       estiver com problemas.

.. _`Baixe o código`: http://www.djangoproject.com/download/
.. _pergunte: http://www.djangoproject.com/community/

Como eu corrijo o erro "install a later version of setuptools"?
---------------------------------------------------------------

Apenas execute o  script ``ez_setup.py`` na distribuição do Django.

Quais são os pré requisitos para se executar o Django?
------------------------------------------------------

O Django requer Python_ 2.3 ou superior. Nenhuma outra biblioteca Python é requerida
para o uso básico do Django.

Para um ambiente de desenvolvimento -- se você apenas quer experimentar o Django --
você não precisa ter um servidor web separado; o Django vem com o seu próprio
servidor leve de desenvolvimento. Para um ambiente de produção, nós recomendamos
`Apache 2`_ e mod_python_, mas o Django seque a especificação WSGI_ spec, o que
significa que ele pode rodar sobre uma ampla variedade de plataformas de servidores.

Se você quer usar o Django com um banco de dados, que é o caso provável, você irá
também precisar de um servidor de banco de dados. O PostgreSQL_ é recomendado, porque somos
fãs do PostgreSQL, e o MySQL_, `SQLite 3`_, e Oracle_ são suportados também.

.. _Python: http://www.python.org/
.. _Apache 2: http://httpd.apache.org/
.. _mod_python: http://www.modpython.org/
.. _WSGI: http://www.python.org/peps/pep-0333.html
.. _PostgreSQL: http://www.postgresql.org/
.. _MySQL: http://www.mysql.com/
.. _`SQLite 3`: http://www.sqlite.org/
.. _Oracle: http://www.oracle.com/

Eu perco algo se eu rodar o Django com uma versão antiga do Python, com a 2.3, em vez de usar a 2.5, por exemplo?
-----------------------------------------------------------------------------------------------------------------

Não, o Django funciona com qualquer versão do Python superior ao 2.3.

Se você usar uma versão mais nova que a 2.3 obviamente, você poderá
tomar vantagem das novas características do Python no seu próprio código, junto com as melhorias
de velocidade e outras otimizações que têm sido feitas na linguagem Python em si.
Mas o framework Django em si deve funcionar igualmente bem nas versões 2.3, 2.4 ou 2.5 do Python.

Eu tenho de usar mod_python?
----------------------------

Nós recomendamos o uso do mod_python para produção, mas graças ao padrão WSGI_,
você não é obrigado a usá-lo. O Django pode falar com qualquer servidor
compatível com o WSGI. Outras configurações de deployment além do mod_python são
FastCGI, SCGI ou AJP. Veja :ref:`Como usar o Django com, SCGI ou AJP <howto-deployment-fastcgi>`
para mais informações.

Veja também a `página do wiki de opções de configuração de servidores`_ para outras
estratégias de deployment.

Se você quer apenas testar e desenvolver coisas em seu computador local, use o
servidor Web de desenvolvimento que vem com o Django.

.. _WSGI: http://www.python.org/peps/pep-0333.html
.. _página do wiki de opções de configuração de servidores: http://code.djangoproject.com/wiki/ServerArrangements

Como eu instalo o mod_python no Windows?
-----------------------------------------

    * Para o Python 2.4, obtenha o mod_python de `build win32 do mod_python para Python 2.4`_.
    * Para o Python 2.4, verifique o `howto do Django no Windows`_.
    * Para o Python 2.3, obtenha o mod_python de http://www.modpython.org/ e leia
      `Running mod_python on Apache on Windows2000`_.
    * Também, tente esse guia (não específico do Windows) `guide to getting mod_python
      working`_.

.. _`build win32 do mod_python para Python 2.4`: http://www.lehuen.com/nicolas/index.php/2005/02/21/39-win32-build-of-mod_python-314-for-python-24
.. _`howto do Django no Windows`: http://thinkhole.org/wp/django-on-windows/
.. _`Running mod_python on Apache on Windows2000`: http://groups-beta.google.com/group/comp.lang.python/msg/139af8c83a5a9d4f
.. _`guide to getting mod_python working`: http://www.dscpl.com.au/articles/modpython-001.html

O Django executará em hosts compartilhados (como TextDrive ou o Dreamhost)?
----------------------------------------------------------------------------

Veja nossa página de `hosts amigáveis ao Django`_ .

.. _`hosts amigáveis ao Django`: http://code.djangoproject.com/wiki/DjangoFriendlyWebHosts

Eu devo usar a versão oficial ou a versão de desenvolvimento?
--------------------------------------------------------------

Os desenvolvedores do Django melhoram o framework todos os dias e são muito bons
em não enviar código quebrado ao repositório. Nós usamos o código do desenvolvimento
(do repositório Subversion) diretamente em nossos servidores, então nós o
consideramos estável. Com isso em mente, recomentadmos que você use a última
versão do código em desenvolvimento, porque ele geralmente tem mais funcionalidades
e menos bugs que as versões "oficiais".
